OOP vs Functional

OOP

Object Oriented Programming is organized around objects and data rather than 'actions' and logic.

Advantages

  • Reuse of Classes: Classes definitions are reusable by the progam it is originally written for as well as other object oriented programs. They can be reused!
  • New data types: Programmers can make new data types. These data types can be used to define data more clearly
  • inheritance: A class can have subclasses. This makes organization of data and information easier. It reduces development time
    • 						class BoggleBoard

							def initialize(grid)
								@boggle_board=grid
							end

							def create_word(*coords)

							coords.map { |coord| @boggle_board[coord.first][coord.last]}.join("")
							end

							def get_row(row)
								return row>@boggle_board.length-1 ? nil : @boggle_board[row]
							end

							def get_col(col)
								column=[]
								@boggle_board.each do |row|
							    	column.push(row[col])   
								end
								return col>@boggle_board.size-1 ? nil : column
							end

						end

    Functional Programming

    Functional Programming is Hierarchical and it emphasizes evaluating expressions rather than the execution of commands.

    Advantages

  • Concurrency: Order of execution of functions is not important
  • Readability: It is very similar to mathematical functions and is therefore more readable

    • Disadvantages

  • Lack of side effects: Side effects means that the execution of a function can change the state of some other function or the outside world. Side effects are the most common way that a program interacts with users or external systems
