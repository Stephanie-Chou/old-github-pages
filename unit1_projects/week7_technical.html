SQL Injection and Prevention

What is SQL Inject?

SQL injection is a way of attacking a database. The malicious DQL code is intended to fool the application and convince it that it wants to run the malicious code which will cause unintended results. For example, the original code may look like 

SELECT first_name FROM directory WHERE last_name='chang' ';
Here, 
'chang'
is the intended input but there is an extra closing quotation mark which leaves the code open for exploitation. A SQL injection would take advantage of the extra quotation and write: 
SELECT first_name FROM director WHERE last_name='chang' or '1' ='1';
Here, the users input is 
'chang' or '1'='1
This will always read as a true statement and therefore, no matter what- that line of SQL code will return whatever the malicious code is seeking.

SQL injection attacks are the most common kind of attack because of their simplicity and because of the desirable nature of sensitive data.

SQL injection prevention

  • Limit or eliminate all user input based queries: User input feilds are a prime opportunity for SQL injection because attackers can write SQL code in an input field
  • Stored Procedures:
  • input validation techniques
  • Verify users and ensure the right people have the right permissions to be making database changes
